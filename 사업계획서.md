# 슥(SSG) 사업계획서

웹소설 IP를 AI 애니메이션 웹드라마로 자동 변환·유통하는 올인원 콘텐츠 플랫폼

작성일: 2026-06-27 · 웹사이트: (추후 기재)

---

## 1. 요약(Executive Summary)

슥(SSG)은 웹소설을 읽고 쓰는 단계에 머무르던 기존 플랫폼을 넘어, 작가의 텍스트가 AI를 거쳐 애니메이션 웹드라마로 자동 변환되고, 독자가 "읽기"와 "보기"를 한 화면에서 교차 소비하는 선순환 생태계를 지향한다.

핵심 가치 사슬은 다음과 같다.

> 창작(Creation) → 영상화(Drama) → 소비(Consumption) → 참여(Engagement) → 확산·재유입(Growth)

작가에게는 제작비 부담 없는 영상화 도구를, 독자에게는 읽던 소설을 드라마로 감상하는 확장된 몰입 경험을, 플랫폼에게는 IP 기반 수익 다각화와 글로벌 유통 가능성을 제공한다.

---

## 2. 문제 정의(Problem)

기존 웹소설 플랫폼 유저 반응 조사에서 도출한 페인포인트(pain point)는 다음과 같다.

| 영역 | 핵심 불만 |
| :--- | :--- |
| 탐색·추천 | 프로모션이 랭킹을 오염시켜 진짜 인기작 구분 불가, 광고성 추천과 실제 추천 혼재 |
| 광고 | 가상캐스팅(배우 연기) 광고에 압도적 거부감 — "원작 이미지와 불일치", "몰입 깨짐" |
| 뷰어 | 표지 확대 불가, 기본 커스터마이징 부재, 어색한 티티에스(TTS) 침묵 |
| 결제·서재 | 앱 내 결제 차단(웹 이동 강제), 연재작 서재 분류 초기화 |
| 영상화 | 웹소설 IP의 영상화 진입 장벽이 높아 극소수 흥행작만 제작됨 |

표지 영향력 조사에서, 표지 일러스트는 구매 결정의 1차 필터이자 독자의 캐릭터 상상을 고정하는 핵심 자산임이 확인됐다. 그러나 영상화로 가는 경로는 웹툰화·외주 제작 등 고비용·고리스크 구조에 막혀, IP 가치 대부분이 텍스트 단계에 잠겨 있다.

## 3. 시장 방향성(Market Direction)

### 3.1 거시 흐름

- 웹소설·웹툰 시장은 성숙기에 접어들었고, 경쟁 축이 "연재량"에서 "IP의 멀티 유즈(multi-use)"로 이동하고 있다.
- 나 혼자만 레벨업(글로벌 143억 뷰), 전지적 독자 시점(영화화) 등은 "웹소설 → 웹툰 → 애니 → 드라마 → 게임"의 풀 IP 스택(full IP stack)이 단일 원작 대비 수십 배 가치를 창출함을 입증했다.
- 다만 이 모델은 흥행 상위 0.1% 작품만 누리는 구조다. 영상화 비용과 제작 기간이 진입 장벽이기 때문이다.
- 생성형 AI 영상 도구(클링·런웨이·비오 등)의 급격한 품질 향상으로, 영상 제작 단가가 급락하는 변곡점에 진입했다.

### 3.2 슥(SSG)이 노리는 변화

영상화의 한계비용이 0에 수렴하면, "상위 0.1%만 영상화되던 시장"이 "중·하위 IP까지 영상화되는 롱테일(long-tail) 시장"으로 확장된다. 슥(SSG)은 이 전환의 인프라(infra) 레이어를 선점하는 것을 목표로 한다.

### 3.3 시장 진입 단계(3-Wave 전략)

| 단계 | 시장 | 전략 |
| :--- | :--- | :--- |
| Wave 1 | 신인·중소 작가 IP | 영상화 기회를 못 받던 롱테일 작가에게 무비용 영상화 도구 제공으로 공급 확보 |
| Wave 2 | 독자·시청자 | 교차 소비 경험으로 리텐션 확보, 유료 전환 |
| Wave 3 | 기성 IP·글로벌 | 검증된 파이프라인으로 대형 플랫폼 IP 영상화 수주, 다국어 유통 |

### 3.4 확장 벡터(Expansion Vectors)

본 사업의 본질은 "텍스트 IP를 영상으로 자동 변환하는 엔진"이며, 이 엔진은 웹소설에 국한되지 않고 다음 방향으로 확장된다.

| 축 | 확장 경로 | 확장 논리 |
| :--- | :--- | :--- |
| 콘텐츠 | 웹소설 → 웹툰·동화·시나리오·게임 스토리 | 동일 변환 엔진을 인접 텍스트 IP에 재사용 |
| 언어·지역 | 국내 → 영어권·일본·동남아 | 다국어 립싱크·더빙 자동화로 현지화 한계비용 최소화 |
| 포맷 | 웹드라마 → 숏폼·예고편·굿즈·인터랙티브 | 단일 IP에서 파생 포맷 다중 생산(멀티 유즈) |
| 사업 모델 | B2C 플랫폼 → B2B 영상화 솔루션(API) | 외부 플랫폼·스튜디오에 변환 엔진을 SaaS로 제공 |

핵심은 "한 번 검증된 파이프라인이 콘텐츠 종류·언어·포맷을 넘나들며 재사용된다"는 점이다. 따라서 보수적 기본 시나리오(아래 7장)에 비해 상방 잠재력(upside)이 구조적으로 크다.

## 4. 경쟁 분석(Competitive Analysis)

### 4.1 포지셔닝 맵

```
                    자동화 높음
                        ↑
       애니메이트에이아이  |   [슥(SSG) 목표]
       픽토리            |   (웹소설 특화 + 완전 자동)
                        |
 일반 영상 ←────────────┼────────────→ 웹소설·애니 특화
                        |
       런웨이 / 클링      |   노벨에이아이
       (엔진)           |   (이미지만)
                        ↓
                    자동화 낮음
```

목표 사분면(자동화 높음 × 웹소설 특화)은 현재 비어 있다.

### 4.2 경쟁 그룹별 분석

| 구분 | 대표 서비스 | 강점 | 한계 | 슥(SSG)의 차별점 |
| :--- | :--- | :--- | :--- | :--- |
| 웹소설 플랫폼 | 네이버 시리즈, 카카오페이지, 왓패드(Wattpad) | 압도적 IP 보유·트래픽 | 영상화는 외주·고비용, 작가 직접 영상화 불가 | 원작 소싱 협업 + 자체 영상화 자동화 |
| 범용 AI 영상 | 런웨이(Runway), 클링(Kling), 비오(Veo) | 영상 품질·범용성 | 웹소설 워크플로 없음, 캐릭터 일관성 수동 | 엔진으로 내재화, 웹소설 전용 파이프라인 제공 |
| AI 스토리 영상화 | 애니메이트에이아이(AnimateAI), 픽토리(Pictory) | 텍스트→영상 자동화 근접 | 웹소설 장르 문법 미반영, 수익 분배 부재 | 장르 특화 각색 AI + 작가 수익 쉐어 |
| 창작·이미지 통합 | 노벨에이아이(NovelAI) | 창작 보조 + 애니 이미지 | 영상·유통 미지원 | 영상 자동화·스트리밍·유통까지 수직 통합 |

### 4.3 진입 장벽 및 방어 전략(Moat)

- 데이터 해자: 작가가 검수한 "텍스트 → 스크립트 → 영상" 정합 데이터가 누적될수록 각색 AI 품질이 경쟁사가 복제하기 어려운 수준으로 고도화된다.
- 양면 네트워크 효과: 작가 공급이 많을수록 시청 콘텐츠가 늘고, 시청자가 많을수록 작가 수익이 커지는 양면 시장(two-sided market) 구조.
- 전환 비용: 작가의 세계관·캐릭터 프리셋·수익 이력이 플랫폼에 락인(lock-in)된다.

### 4.4 핵심 차별화 요소

1. 장르 문법(로맨스판타지·무협 등)을 이해하는 웹소설 전용 씬 분할·각색 AI
2. 에피소드 전체에서 동일 캐릭터를 보장하는 일관성 유지 파이프라인
3. 조회수 기반 작가 수익 분배 모델
4. 원작 소설 ↔ 드라마 교차 소비(읽기와 보기를 한 플랫폼에서 연결)
5. 크런치롤·넷플릭스 등 글로벌 스트리밍 유통 연계

## 5. 솔루션 및 서비스 구조(Solution)

슥(SSG)은 네 개 레이어(layer)로 구성된다.

| 레이어 | 명칭 | 주요 역할 | 핵심 가치 |
| :--- | :--- | :--- | :--- |
| Creation | 작가 영역 | AI 보조 집필, 세계관 관리, 표지·삽화 생성 | 창작 효율 |
| Drama | 웹드라마 영역 | 웹소설 → AI 애니메이션 웹드라마 제작·유통 | IP 확장·영상화 자동화 |
| Consumption | 독자 영역 | 뷰어 최적화, 티티에스(TTS), 드라마 스트리밍 | 몰입감·접근성 |
| Engagement | 커뮤니티 | 팬아트, 2차 창작, 비하인드 공유 | 팬덤 형성·확산 |

작가 도구는 스마트 에디터(자동 완성·문체 유지), 월드 빌딩 매니저(설정 오류 체크), AI 비주얼라이저(삽화 생성), AI 커버 제너레이터(표지 생성)를 제공한다.

웹드라마는 크리에이터 스튜디오(작가 전용)에서 텍스트 업로드 → AI 스크립트 자동 생성 → 작가 검토 → 캐릭터 외형 설정 → 씬 편집 → 게시 플로우를 제공하고, 뷰어 플랫폼(독자·시청자)은 에피소드 단위 스트리밍(5–15분), 원작 구절과 장면의 싱크(sync) 표시, 시네마틱(cinematic) UI를 차별점으로 한다.

독자 사이드는 패드 환경 최적화 반응형 웹으로 구현하며, 랜딩(Landing)은 애플 뮤직(Apple Music) 스타일 3D 캐러셀(carousel)로 카테고리 정체성을 전달한다.

## 6. AI 제작 파이프라인(Pipeline)

비용 효율과 캐릭터 일관성을 최우선으로 설계했다.

| 단계 | 도구 | 비용 | 자동화 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| 1 스크립트 | 제미나이(Gemini) 2.0 플래시 | 무료 | 보조 | 씬 분할·각색 AI 제안 후 수정 |
| 2 캐릭터 레퍼런스 | 미드저니(Midjourney) | 별도 | 수동 | 직접 생성 후 업로드 |
| 3 씬 이미지 | 레오나르도(Leonardo.ai) | 유료 | 자동 | 레퍼런스로 캐릭터 일관성 유지 |
| 4 티티에스(TTS) | 제미나이(Gemini) 2.5 플래시 | 무료 | 자동 | 캐릭터별 목소리 프리셋 고정 |
| 5 영상 합성 | 클링(Kling) 3.0 | 월 $6.99 | 자동 | 이미지+오디오 → 영상 |

MVP 단계에서는 영상 합성을 제외하고 이미지 슬라이드쇼(slideshow) + 티티에스 오디오 형식으로 먼저 출시해 단가를 최소화한다.

## 7. 비즈니스 모델 및 예상 수익(Business Model & Revenue)

### 7.1 수익원

| 모델 | 대상 | 내용 | 성격 |
| :--- | :--- | :--- | :--- |
| 유료 회차 결제 | 독자 | 소설 유료 회차 단건 결제 | 단기 현금흐름 |
| 드라마 구독 | 시청자 | 월정액 무제한 시청 | 반복 매출(MRR) |
| 크리에이터 크레딧 | 작가 | AI 생성 횟수·시간 기준 차감 | 사용량 기반 |
| 수익 쉐어 | 작가 | 조회수 기반 분배(플랫폼 수수료) | 거래 수수료 |
| IP 라이선싱 | 플랫폼 | 인기작 외부 판권 중개 | 중장기 고마진 |
| 브랜드 광고 | 플랫폼 | 커뮤니티 내 광고(가상캐스팅 배제) | 보조 매출 |

### 7.2 핵심 단가 가정(Unit Economics)

| 항목 | 가정값 |
| :--- | :--- |
| 드라마 구독 ARPU(인당 월 매출) | 9,900원 |
| 유료 독자 ARPU(월) | 6,000원 |
| 유료 전환율(MAU → 결제) | 5% |
| 에피소드당 AI 제작원가 | 2,000~5,000원 |
| 작가 수익 쉐어 비율 | 콘텐츠 매출의 60% 분배 / 40% 플랫폼 |
| 결제 수수료 등 변동비 | 매출의 약 10% |

### 7.3 3개년 매출 시나리오(Conservative Base Case)

아래는 보수적 기본 시나리오다. 마케팅·작가 공급·확장 벡터(3.4) 실현 시 상방 잠재력은 이를 크게 상회한다.

| 구분 | 1년차 | 2년차 | 3년차 |
| :--- | ---: | ---: | ---: |
| 월간 활성 이용자(MAU) | 2만 | 12만 | 50만 |
| 유료 전환율 | 3% | 4% | 5% |
| 유료 이용자 | 600명 | 4,800명 | 2.5만명 |
| 합산 ARPU(월, 결제·구독 혼합) | 7,000원 | 7,500원 | 8,000원 |
| 콘텐츠 매출(연) | 약 0.5억 | 약 4.3억 | 약 24억 |
| IP 라이선싱·광고(연) | 0 | 0.5억 | 4억 |
| 총매출(연) | 약 0.5억 | 약 4.8억 | 약 28억 |
| 플랫폼 귀속 매출(수익 쉐어 후) | 약 0.2억 | 약 2.2억 | 약 13.6억 |

> 산식 예: 3년차 콘텐츠 매출 = 2.5만명 × 8,000원 × 12개월 ≈ 24억. 플랫폼은 콘텐츠 매출의 40% + 라이선싱·광고 전액을 귀속.

### 7.4 수익 구조 단계 전략

- 1년차: 작가 공급 확보가 최우선. 크레딧·구독 매출은 손익분기보다 생태계 활성화 지표로 관리.
- 2년차: 드라마 구독을 반복 매출(MRR)의 축으로 키우고 유료 전환율 개선.
- 3년차: 검증된 흥행 IP의 라이선싱·글로벌 유통으로 고마진 매출 비중 확대.

## 9. 구현능력 및 실현 가능성(Feasibility)

### 9.1 자체 제작이 아닌 조합·오케스트레이션 구조

슥(SSG)은 영상 생성 모델을 직접 개발하지 않는다. 이미 상용화된 검증 엔진(제미나이·레오나르도·클링 등)을 API로 조합하고, 그 사이의 워크플로(텍스트 분할 → 캐릭터 고정 → 씬 생성 → 합성)를 오케스트레이션(orchestration)하는 것이 핵심 기술이다. 따라서 초기 기술 리스크와 개발 난이도가 낮다.

### 9.2 단계별 구현 난이도

| 기능 | 난이도 | 근거 |
| :--- | :--- | :--- |
| 텍스트 → 스크립트 변환 | 낮음 | 범용 LLM 프롬프트(prompt)·파인튜닝으로 구현 가능, 즉시 검증됨 |
| 캐릭터 일관성 유지 | 중간 | 레퍼런스 고정 기반 이미지 생성으로 상용 도구가 이미 지원 |
| 씬 이미지 → 영상 합성 | 중간 | 클링 등 API로 자동화, MVP는 슬라이드쇼로 우회 가능 |
| 플랫폼·뷰어·스트리밍 | 낮음 | 표준 웹 스택(리액트·넥스트·HLS)으로 구현, 신기술 불필요 |

### 9.3 비용 효율 검증

MVP는 영상 합성 단계를 제외하고 이미지 슬라이드쇼 + 티티에스 오디오로 출시한다. 이 구성의 직접 원가는 에피소드당 수천 원 수준이며, 무료 엔진(제미나이 스크립트·티티에스) 비중이 높아 초기 운영비 부담이 작다. 즉, 최소 자본으로 시장 검증이 가능하다.

### 9.4 현재까지의 진척

서비스명 슥(SSG)과 반응형 웹 형태가 확정됐고, 독자 화면(Landing·Preferences·Discovery·Novel Detail·Viewer·Event) 명세와 제작 파이프라인 설계, 경쟁·유저 리서치가 완료된 상태다. 즉 기획·설계 단계를 지나 프로토타입 구현 진입 단계에 있다.

### 9.5 실현 가능성 요약

- 핵심 기술이 "신규 모델 개발"이 아닌 "검증된 엔진의 조합"이므로 기술 리스크가 낮다.
- MVP가 무료·저가 엔진 중심이라 최소 자본으로 출시·검증할 수 있다.
- 표준 웹 스택 기반이라 인력 확보·유지보수 부담이 작다.
- 엔진이 교체·고도화되면 별도 개발 없이 품질이 자동 상승한다(외부 기술 발전의 수혜 구조).
